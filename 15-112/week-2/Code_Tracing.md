# Code Tracing

*Code Tracing(代码跟踪)*和*Reasoning Over Code(代码推理)*，主要是训练：**静态理解程序运行过程的能力（Mental Execution）**。

## 建立调用栈思维

理解函数如何一层层调用。所有方法函数和变量的定义语句在没有被调用之前，就是存储在内存中的一个数据块，你只要关注实际启动或运行时，有哪些方法函数和变量被调用执行了。然后根据之前学习的执行顺序，确认整个执行流程中涉及的函数和变量以及它们的执行顺序，这就是调用栈思维。

## 建立变量状态变化模型

调用栈的逻辑建立后，我们还需要考虑到变量的状态变化，每个方法函数都是对变量的一次处理或加工，这些方法函数也会输出一个值，这个值可能就是下一个方法函数的入参，变量再经过方法函数的处理后都可能发生变化，数据类型的变化（`int -> bool`），需要注意的是对于函数的局部变量的改变只影响当前帧的计算，对于*list*的修改需要慎重，这涉及指针对象的修改。

## 建立控制流模型

我觉得这是对调用栈思维的补充，增加了一些流程控制语法（`if/for/while/return`）后，真实情况下的完整的调用栈。

## 建立逆向推理能力

主要是在*debug*和排错阶段需要，你需要根据程序反馈的错误信息或日志进行反推是哪个函数处理逻辑出现问题导致的。
